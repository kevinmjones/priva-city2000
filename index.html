02 How we build

Design

Start with the simplest interface that solves the problem. Strip away everything that is not essential.

Build

Write clean code that runs offline first. No server dependencies, no hidden data collection.

Ship

Release when it works, not when it is perfect. Iterate based on real use, not imagined requirements.

Listen

Stay close to users. Every feature must prove itself against the question: does this make the tool better?
